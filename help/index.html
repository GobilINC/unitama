Start building your website

OK, you have all the files. What's next? Well, there are two ways. The first one is to upload those files to your hosting and make all changes via FTP. The second one is to build your website locally (read more about making changes in next chapters) and upload it to your hosting later.

It's completely up to you which way to choose, but my suggestion is to follow the second way if you are not a professional web developer.

Anyway, sooner or later, you'd have to upload those files to Document Root directory of your server (typically it's public_html or www folders). Which files should you upload? Here's the list:

  • [template-name].html — main HTML file of your choice from the root directory of the template. You should rename it to index.html
  • feedback.html — a script that handles your Contact Form requests.
  • /assets/js/ — complete the folder with JavaScript files.
  • /assets/css/ — complete folder with CSS files.
  • /assets/libs/ — complete folder with PHP libraries.
  • /assets/fonts/ — complete folder with font files.
Optionally you can upload the /assets/less/ folder if you are going to compile LESS files on your web server.
It's very important to keep the file structure in the same way as it is. If you want to change it you should also change paths to these files in your HTML file in accordance with their new locations.

Bootstrap 3 Framework

This template is built upon a Bootstrap 3 — the most popular HTML, CSS, and JS framework for developing responsive, mobile first projects on the web.

You can use almost all framework features in your project with a few limitations:

  • The layout of page doesn't have navigation bar in the top of the page.
  • The whole template consists of slides (screens of information) with the transitions between so you should keep it in mind and not overload each slide.
If you are not familiar with the framework, please check Bootstrap 3 website for the complete documentation.

Color schemes

OK, now is the fun part. This template includes several different color schemes, so you should know how to switch between them.

To change the color scheme for the entire page you have to link your page with the right CSS file. You can find those files in /assets/css/ folder. Here is an example of how you should add them into your page:

<head>
   ...
   <!-- Include stylesheets with 'Dark' color scheme -->
   <link rel="stylesheet" href="assets/css/bootstrap-dark.css" />
   ...
</head>

Creating slides

As you've already noticed the layout of Zbz! Spash consists of a several slides with animated transitions between them. Although the animations are created automatically, you still have to follow the basic pattern of HTML markup to make it work.

<!-- Slide keeper -->
<div class="zbz-slide-keeper" >

    <!-- First slide -->
    <section class="zbz-slide" data-nav="Home">
        <div class="zbz-slide-inner">
            <div class="zbz-slide-content zbz-on-top">
            
                <!-- Your content here -->
                
            </div><!-- /.slide-content -->
        </div><!-- /.slide-inner -->
    </section>
    
    <!-- Second slide -->
    <section class="zbz-slide" data-nav="Second slide">
        <div class="zbz-slide-inner">
            <div class="zbz-slide-content zbz-on-top">
            
                <!-- Your content here -->
                
            </div><!-- /.slide-content -->
        </div><!-- /.slide-inner -->
    </section>

</div><!-- /.slide-keeper -->

There are a few basic elements. <section class="zbz-slide" /> is an element which contains every screen of a webpage. All slides are wrapped in <div class="zbz-slide-keeper" /> element. In fact this is a general structure that should be kept untouched. But there are a few more elements that helps your website to look better. <div class="zbz-slide-inner" /> and <div class="zbz-slide-content" /> elements are responsible for keeping the content vertically aligned in the center of the screen.

Navigation

As you've noticed that each <section class="zbz-slide" /> has a data-nav attribute. The value of this attribute would be used as a name of the slide in page navigation. In order to create the navigation (I'm talking about those bullets on the right side of a page) all you have to do is to set those attributes for your slides and it would be created automatically.

<section class="zbz-slide" data-nav="Slide title">
    <!-- Slide content -->
</section>

If you don't want a specific slide to be in the navigation simply remove the attribute for this slide.

<section class="zbz-slide">
    <!-- Slide content -->
</section>

Responsiveness

Bootstrap includes a responsive, mobile first fluid grid system that appropriately scales up to 12 columns as the device or viewport size increases. It includes predefined classes for easy layout options.

You can find more information about working with Bootstrap grid system in documentation.

Zbz! Splash follows the same principles and provides classes for setting feature boxes sizes targeted to different viewport sizes.

Scaling classes apply to devices with screen widths greater than or equal to the breakpoint sizes, and override scaling classes targeted at smaller devices. Therefore, applying any .zbz-md- class to an element will not only affect its styling on medium devices but also on large devices if a .zbz-lg- class is not present.

Additionaly Zbz! Splash disables animations for small and extra small viewports to increase the speed of page rendering.

Transitions between the screens

On scroll animations with the circles and appearance of content parts are created automatically. The only thing you have to do is to tell the script which elements of the page should be those flying elements and how should the other parts appear and disappear.

Flying circles

Let's say you design a slide with some text and a feature box. And you want an icon of that feature box to be the one of those circles flying over the screen. All you have to do is to set an attribute data-role="zbz-flying" to an element that you want to take place in animation. You should also wrap the content of .zbz-circle in a .zbz-inner to make it disappear when current slide is inactive. So your code would look like this:

<!-- This is your slide -->
<section class="zbz-slide">
    <div class="zbz-slide-inner">
        <div class="zbz-slide-content">
        
            <!-- Feature box -->
            <div class="zbz-feature-side zbz-xs-2x">
            
                <!--
                    Add data-role="zbz-flying" attribute
                    to make this circle animated
                -->
                <div class="zbz-circle" data-role="zbz-flying">
                    <!--
                        Wrap the content of .zbz-circle in a .zbz-inner
                        to make it disappear when current slide is inactive
                    -->
                    <div class="zbz-inner">
                        <i class="fa fa-rocket"></i>
                    </div>
                </div>
                
                <!-- Description -->
                <div class="zbz-description">
                    The description goes here.
                </div>
            </div>

        </div>
    </div>
</section>

Appearing and disappearing of content

Now the circle beautifully flies in and out but the description stays untouched. To change that you should set data-appear and data-disappear attributes for <div class="description"> as long as for other visible elements of the slide. There is also an optional data-delay attribute which is responsible for delay in the appear and disappear transitions.

Here is how it looks in the code:

<!-- This is your slide -->
<section class="zbz-slide">
    <div class="zbz-slide-inner">
        <div class="zbz-slide-content">
        
            <!-- Feature box -->
            <div class="zbz-feature-side zbz-xs-2x">
                <div class="zbz-circle" data-role="zbz-flying">
                    <div class="zbz-inner">
                        <i class="fa fa-rocket"></i>
                    </div>
                </div>
                
                <!--
                    Add data-appear and data-disappear attributes
                    to make an element nicely appear and disappear
                -->
                <div class="zbz-description" data-appear="slideInUp" data-disappear="slideOutUp" data-delay="10">
                    The description goes here.
                </div>
            </div>

        </div>
    </div>
</section>

Here is a table of available appearing and disappearing effects:

Appear Disappear Description
fadeIn fadeOut Fading
slideInUp slideOutUp Slide from down to up
slideInDown slideOutDown Slide from up to down
slideInLeft slideOutLeft Slide from right to left
slideInRight slideOutRight Slide from left to right
scrollIn scrollOut Scroll normally

CSS/LESS Structure

This template's CSS is built on LESS, a preprocessor with additional functionality like variables, mixins, and functions for compiling CSS. So the best way to make changes in styles is to edit LESS files in /assets/less/ folder.

The template has Bootstrap 3 framework in it's core, which means that if you are familiar with it then you know how to customize it. If you are not, then you can find a complete documentation on Bootstrap 3 framework website.

Also there are some custom classes included. You can find them in /assets/less/custom/ folder.

Which files to compile?

The main files for compilation are bootstrap-[color-scheme].less in /assets/less/. After making changes in any file that is imported to main file you should compile it to bootstrap-[color-scheme].css in /assets/css/.

Please note that for each color scheme there is a separate bootstrap-*.less LESS file which imports the specific zbz-color-scheme-*.less file.

Contact form

This template comes with a working AJAX contact form. It includes three major components:

  1. JavaScript which handles form submission and processing the response from server /assets/js/zbz-feedback.js
  2. PHP script which handles AJAX request from the client and sends an email /feedback.php
  3. Swift Mailer library in /assets/libs/SwiftMailer/

Setting things up

Make sure that JavaScript file is included in your HTML:

<script type="text/javascript" src="assets/js/zbz-feedback.js"></script>

It is also important to set right attributes for your form. First of all it should have a data-role="zbz-feedback" attribute to make it detectable by JavaScript. It should also have a zbz-feedback-form class applied for the right styling. And action attribute should point to a PHP script (normally action="feedback.php" if you don't rename it).

<form class="form zbz-feedback-form" role="form" data-role="zbz-feedback" method="POST" action="feedback.php">

Now you have to adjust a few parameters in /feedback.php. Set the recipient's (yours) e-mail address and name. Subject and message 'before' and 'after' parts. Choose the type of transport and provide transport specific settings. Here is an example for sending e-mails using your gmail account:

// Recipient e-mail
$TO_EMAIL = 'youremail@gmail.com';

// Recipient name
$TO_NAME = 'Your Name';

// Subject
$SUBJECT = 'Message from yoursite.com';

// Body before
$BODY_BEFORE = '<p>You have a message:</p><hr/>';

// Body after
$BODY_AFTER = '<hr/><p>This message was sent via contact form on your site.</p>';

// Transport (available transports are: 'smtp', 'sendmail' and 'mail')
$TRANSPORT = 'smtp';


// SMTP Settings

// SMTP host
$SMTP_HOST = 'tls://smtp.gmail.com';

// SMTP port
$SMTP_PORT = 465;

// SMTP username
$SMTP_USERNAME = 'youremail@gmail.com';

// SMTP password
$SMTP_PASSWORD = 'Y0urPa55w0rd';


// Sendmail settings

// Linux command to run Sendmail
$SENDMAIL_RUN = '/usr/sbin/sendmail -bs';
